**Requires:** a signed-in account. Publishing a snapshot additionally requires current publishing permission.

Open [Publishing](/#/publishing). If you cannot yet publish, the page shows **Apply to publish**. Add an optional message and submit the application. An open application remains visible while it awaits maintainer review.

![The Publishing page showing the public snapshot editor available to an approved Publisher.](/docs/screenshots/publishing.png)

## How permission works

A maintainer can approve an application, which currently unlocks publishing in the app. This is separate from a durable system Publisher role; either an approved application or an appropriate seeded role can allow publishing.

## Publish a snapshot

When publishing is available:

1. Enter a public **Title** and unique **Slug**.
2. Add an optional description.
3. Optionally enter a Publish filter to limit the resolved list snapshot.
4. Review the number of Season rows that will be included.
5. Select **Publish snapshot** and wait for all writes to be acknowledged.

Publishing stores a snapshot, not a live view of every subsequent personal edit. Publishing the same owned slug again creates the next snapshot version. Your published lists are shown on the same page and are discoverable through [Published Lists](/#/published).

Create a display name, public slug, bio, and public-profile setting from your email link in the app header. Public profiles can show their owner's published lists.
