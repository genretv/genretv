# Derived Schedule Placement From Release Windows

Schedule View placement and ordering will be derived from the full local List, Release Windows, Date Confidence, Show Lifecycle Status, and the current date rather than fixed by a manually assigned section. The stored Source Schedule Section preserves imported or maintainer intent and remains a fallback when release evidence cannot determine placement; it is not the effective Schedule Section displayed to readers.

The effective sections, in display order, are Now Showing, Upcoming, Awaiting Renewal or Cancellation, and Finished. Every greenlit Season is a row and remains Upcoming until confirmed release evidence says it has started; multiple future Seasons for one Show all appear. Confirmed releases that have started and not concluded are Now Showing. When no current or future Season exists, the latest concluded Season represents an open Show in Awaiting Renewal or Cancellation or a terminal Show in Finished. Older Seasons are historical and omitted from the Schedule View. A Show can appear in Now Showing and Upcoming simultaneously.

Exact dates, months, Release Seasons, and year-only values can all participate in placement and ordering while preserving their original precision and Date Confidence for display and filtering. An imprecise period uses its inclusive end as its ordering boundary: the last day of a month, the last day of a Northern Hemisphere meteorological Release Season, or December 31 for a year. An exact date covered by an imprecise period always appears before that period in either chronological or reverse-chronological views. This presents known dates before estimates without moving the broad period ahead of entries outside its bounds. Unknown windows remain undated and sort after usable timing rather than receiving an inferred year or synthetic date.

When the source omits a year from a recognized month, date, or Release Season because the page supplies chronological context, the source update date anchors that year inference. Estimated windows order Upcoming Seasons but never make them Now Showing merely because the estimate passes.

Schedule View defaults to chronological `When` ordering, with releases lacking usable timing after dated releases. The Blogspot table row position is extraction metadata only: it is neither persisted in canonical, personal, or published Season rows nor exposed as a Schedule ordering option.

Release Pattern distinguishes weekly, bulk, and unknown Seasons. Bulk streaming releases stay Now Showing for a Current Grace Period, initially five weeks. Blogspot imports from Netflix default to bulk; a parsed finale date instead identifies a normal non-bulk run. A complete set of dated Episode rows can supply the finale when the Season has no explicit finale date. For an imported bulk row recorded as current without a release date, the source update date is the conservative start of that grace period.

In Now Showing, bulk releases display `Binge`. A non-bulk release with an exact start date displays the weekday derived from that date and retains any known finale, matching the recurring schedule readers need once the premiere has passed.
